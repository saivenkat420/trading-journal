═══════════════════════════════════════════════════════════════════
                        KEY DATA FLOWS
═══════════════════════════════════════════════════════════════════

1. AUTHENTICATION FLOW:
   User → Frontend → POST /api/auth/login → Backend → Query DB
   → Generate JWT → Response → Frontend stores token

2. TRADE CREATION FLOW:
   User fills form → Frontend → POST /api/trades (FormData)
   → Backend validates → DB Transaction (INSERT trade, accounts, files)
   → Response → Frontend redirects

3. FILE UPLOAD FLOW:
   User selects files → Frontend → FormData → Backend (Multer)
   → Convert to Buffer → DB INSERT (BYTEA) → Response

═══════════════════════════════════════════════════════════════════
                    KEY TECHNOLOGIES
═══════════════════════════════════════════════════════════════════

Frontend:  React 18, TypeScript, Vite, Axios, Tailwind CSS
Backend:   Node.js, Express.js, JWT, Joi, Multer, Winston
Database:  PostgreSQL (Neon), pg library
Deploy:    Vercel (Frontend), Render (Backend)
Auth:      JWT tokens, localStorage
Storage:   Files stored as BYTEA in PostgreSQL

═══════════════════════════════════════════════════════════════════
                    KEY DESIGN DECISIONS
═══════════════════════════════════════════════════════════════════

✓ 3-Tier Architecture (Separation of concerns)
✓ RESTful API (Stateless, scalable)
✓ JWT Authentication (No server-side sessions)
✓ Files in Database (Simplicity, consistency)
✓ PostgreSQL (ACID compliance for financial data)
✓ Connection Pooling (Database efficiency)
✓ Transaction Management (Data consistency)
